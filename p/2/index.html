何時執行整合測試？

通常整合測試執行時間比單元測試長很多，也因為整合測試的執行頻率無法像單元測試一樣頻繁，比較適合每日排程執行。 為了在進行開發而頻繁執行測試時，不需要等待整合測試執行，所以特別將整合測試獨立成一個 Source Set，不和單元測試程式放在一起。

另一種方式是將整合測試獨立成一個專案，在 Gradle 設定上應該會比較簡單 XD。 以下說明是針對比較搞剛的 Gradle Source Set 1。

建立容器化 Spring Boot 專案

發布： 2021-04-30   |   分類： 技術   |   閱讀： 222 字 ~1 分鐘

建立新 Spring Boot 專案

  1. 使用 Intellij IDEA 建立新專案，專案資訊如下：

    • Spring Boot 版本：2.4.5
    • 專案類型：Gradle
    • 語言：Java 11
    ① 使用 Intellij IDEA 內的 Spring Initailizr 創建新專案② 加入 Spring Web Dependency

  2. 增加 HelloController.java 如下：

    package cc.secondbrain.demo.controller;
   
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
   
@RestController
public class HelloController {
   
    @RequestMapping("/")
    public String hello() {
        return "Hello Spring Boot";
    }
   
}

  3. 啟動 Spring Boot 服務，用瀏覽器瀏覽 http://localhost:8080/，就可以看到「Hello Spring Boot」

本篇利用 Hugo 建立個人化網站，基本應用參考 Hugo 系列文：

  • Hugo 教學 – 基礎篇
  • Hugo 教學 – 進階篇

Custom Domain

  1. 在 GoDaddy 購買屬於自己的網域，例如：2ndbrain.cc
  2. 在 Terminal 用指令 dig 查詢個人 GitHub Pages 的 IP
    dig {YOUR_USER_NAME}.github.io
  3. 在 GoDaddy 設定 IP 和 轉址 Domain
  4. 在 Repository {YOUR_USER_NAME}.github.io 的 Settings >> Pages 設定 Custom domain
  5. 在 Repository {YOUR_USER_NAME}.github.io 的根目錄加入 CNAME 檔案
    echo 'YOUR_DOMAIN' >> CNAME

    echo '2ndbrain.cc' >> CNAME

Hugo 教學 -- 基礎篇

發布： 2021-04-11   |   更新： 2021-04-19   |   分類： 技術   |   閱讀： 379 字 ~1 分鐘

本篇利用 Hugo 從無到有建立個人網站，進階應用參考 Hugo 系列文：

  • Hugo 教學 – 基礎篇
  • Hugo 教學 – 進階篇

Get Started 1 2

  1. 在 GitHub 建立一個 Private Repository hugo-site 管理網站原始碼
  2. 在 GitHub 建立一個 Public Repository {YOUR_USER_NAME}.github.io 用來上傳靜態網頁到 GitHub Pages
  3. 下載 hugo-site 到 Local
    git clone https://github.com/{YOUR_USER_NAME}/hugo-site.git

    或下載包含 Git Submodule 的 Repository 到 Local

    git clone --recursive https://github.com/{YOUR_USER_NAME}/hugo-site.git
  4. 使用 Homebrew 安裝 Hugo
    brew install hugo

    使用 Hugo CLI 確認 Hugo 安裝是否成功

    hugo version
  5. 在放置 hugo-site 的同層路徑（非 hugo-site 資料夾內），使用 Hugo CLI 建立新靜態頁面網站
    hugo new site hugo-site

    若發生以下錯誤：

    Error: /path/to/hugo-site already exists and is not empty. See --force.

    加上參數 --force 強制建立網站

    hugo new site hugo-site --force
  6. 在 themes.gohugo.io 網站樣式，Fork 欲選樣式的 Repository 到自己的 GitHub，以方便調整樣式
  7. 以 hugo-theme-next 樣式 3 為例，將樣式專案以 Submodule 形式加入 hugo-site
    cd hugo-site
git submodule add https://github.com/{YOUR_USER_NAME}/hugo-theme-next.git themes/next
  8. 根據 ./themes/next/config.toml 的設定調整 ./config.toml
  9. 使用 Hugo CLI 建立新文章
    hugo new posts/2021/04/get-started-hugo.md
  10. 使用 Hugo CLI 在 Local 起 Server，建置並在 http://localhost:1313/ 檢視網站更新
    hugo server -D

    檢視新文章 http://localhost:1313/posts/2021/04/get-started-hugo/

  11. 使用 Hugo CLI 在 hugo-site/public 建製靜態網頁
  12. 將 hugo-site/public Push 到 Repository {YOUR_USER_NAME}.github.io 後，在 https://{YOUR_USER_NAME}.github.io/ 檢視網站更新
