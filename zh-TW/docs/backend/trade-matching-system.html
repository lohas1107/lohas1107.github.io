交易撮合系統

撰寫一個基本的交易撮合系統（Trade Matching System）需要考慮多個方面，包括訂單的接收、訂單簿（Order Book）的管理、交易撮合的算法以及成交的通知等。以下是一個使用 Rust 設計的簡化版交易撮合系統的大致框架，演示了如何實現一個基本的撮合引擎。

系統設計要點​

  • 訂單類型：支持市價單（Market Orders）和限價單（Limit Orders）。
  • 訂單簿（Order Book）：分別管理買單和賣單。
  • 撮合算法：基於價格和時間優先原則進行撮合。

Rust 實現步驟​

步驟 1: 定義訂單結構和相關枚舉​

#[derive(Debug, Clone, PartialEq)]
enum OrderType {
    Limit(f64), // 限價單，帶有價格
    Market,     // 市價單
}

#[derive(Debug, Clone)]
enum OrderSide {
    Buy,
    Sell,
}

#[derive(Debug, Clone)]
struct Order {
    id: u64,
    side: OrderSide,
    order_type: OrderType,
    quantity: u64, // 未成交數量
    price: Option<f64>, // 僅對限價單有效
}

步驟 2: 定義訂單簿​

訂單簿用於存儲和管理所有待撮合的訂單。

use std::collections::VecDeque;

struct OrderBook {
    buy_orders: VecDeque<Order>,
    sell_orders: VecDeque<Order>,
}

impl OrderBook {
    fn new() -> OrderBook {
        OrderBook {
            buy_orders: VecDeque::new(),
            sell_orders: VecDeque::new(),
        }
    }

    // 添加訂單到訂單簿
    fn add_order(&mut self, order: Order) {
        match order.side {
            OrderSide::Buy => self.buy_orders.push_back(order),
            OrderSide::Sell => self.sell_orders.push_back(order),
        }
    }

    // 撮合邏輯（簡化版）
    fn match_orders(&mut self) {
        // 此處僅為示例，實際撮合邏輯會更複雜
    }
}
