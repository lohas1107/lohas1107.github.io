整合測試 Rust API

在這個示例中，我將指導你如何使用 warp 框架在 Rust 中創建一個簡單的 "Hello World" API，以及如何為它撰寫一個整合測試。

步驟 1: 添加依賴​

首先，在你的 Cargo.toml 文件中添加 warp 和 tokio 作為依賴，因為 warp 需要 tokio 運行時。另外，為了測試，我們還會使用 reqwest 來發送 HTTP 請求到我們的 API。

[dependencies]
warp = "0.3"
tokio = { version = "1", features = ["full"] }

# 僅用於測試的依賴
[dev-dependencies]
reqwest = "0.11"

步驟 2: 創建 Hello World API​

在 src/main.rs 中創建一個簡單的 warp 服務來響應 "Hello World"。

use warp::Filter;

#[tokio::main]
async fn main() {
    // 創建一個路由
    let hello = warp::path::end()
        .map(|| warp::reply::json(&"hello world"));

    // 啟動服務
    warp::serve(hello)
        .run(([127, 0, 0, 1], 3030))
        .await;
}
備註

warp 的 Filter trait 定義了如何從請求中提取資訊、處理請求並生成響應。這些過濾器可以被組合和重用，使得處理複雜的 HTTP 請求變得更加簡單和高效。 如果沒有加上 use warp::Filter; 會出現以下錯誤訊息。

這段代碼創建了一個簡單的 HTTP 服務，當訪問 http://127.0.0.1:3030 時，它會回應 "hello world"。

步驟 3: 撰寫整合測試​

// src/main.rs
#[cfg(test)]
mod tests {
    #[tokio::test]
    async fn test() {
        let client = reqwest::Client::new();
        let response = client.get("http://localhost:3030")
            .send()
            .await
            .unwrap();

        assert!(response.status().is_success());
        assert_eq!("\"hello world\"", response.text().await.unwrap());
    }
}

這個測試使用 reqwest 發送一個 GET 請求到我們的 API，然後驗證響應狀態碼和響應體。

備註

請注意，此處為了方便，將整合測試也寫在 src/main.rs，常見的作法應該在你的項目根目錄下創建一個名為 tests 的目錄， 並在該目錄中創建一個 Rust 測試文件，比如 hello_world.rs。

運行你的 API 和測試​

  1. 運行 API：在終端中使用 cargo run 命令來運行你的 API。
  2. 運行測試：在另一個終端窗口中，使用 cargo test 命令來運行整合測試。
備註

請注意，整合測試假設你的 API 已經在運行，並且監聽在預設的地址和端口上。如果你的 API 位址或端口不同，你需要在測試中相應地修改 URL。
