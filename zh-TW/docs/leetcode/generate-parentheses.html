22. Generate Parentheses

產生括號組合

在這篇文章中，我們將探討 LeetCode 的「產生括號組合」問題，這是一個經典的問題，測試你對遞迴和回溯法的理解。這個問題非常適合展示不僅僅是算法思維，還有如何構建一個明確定義的問題空間並在其中導航的重要性。

提示
  • 1 <= n <= 8

理解限制條件非常關鍵，因為它告訴我們問題空間的潛在大小，以及我們的解決方案必須操作的限制範圍。

特殊情況​

在處理遞迴時，特別是在生成組合或排列時，考慮基礎案例和遞迴終止的條件至關重要。對於這個問題，特殊情況圍繞著添加左括號或右括號的條件：

  • 只有當數量小於 n 時，才能添加左括號 (。
  • 只有當其數量小於左括號 ( 的數量時，才能添加右括號 )，確保有效性。

這個簡單而有效的檢查確保我們只生成有效的組合，從而有效地導航問題空間。

方法和解決方案​

這個問題本質上建議使用遞迴來探索所有可能的括號組合。然而，不是所有組合都是有效的，因此我們需要一個策略來只生成有效的組合。我們通過維持兩個計數器來做到這一點：left 表示左括號 ( 的數量，right 表示右括號 ) 的數量。在遞迴的任何點，如果沒有超過 n，我們可以添加一個左括號，如果沒有超過左括號的數量，我們可以添加一個右括號。

回溯法​

以下是一個實現上述邏輯的 Python 函數：

class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        result = []

        def backtrack(left: int, right: int, combination: str):
            if len(combination) == 2 * n:
                result.append(combination)
                return
            if left < n:
                backtrack(left + 1, right, combination + "(")
            if right < left:
                backtrack(left, right + 1, combination + ")")

        backtrack(0, 0, "")
        return result

複雜度分析​

生成有效的括號組合的複雜度分析涉及了理解遞迴樹的分支因素和這棵樹的深度：

  • 時間複雜度： O(2^n) - 這是由於卡塔蘭數的結果，它大約代表了正確組合的數量，並且呈指數增長。然而，由於剪枝無效狀態（當 right > left 時），實際的時間複雜度被第 n 個卡塔蘭數限制。
  • 空間複雜度： O(n) - 相對於輸入大小 n，空間複雜度是線性的，由於遞迴棧的深度。對於每次遞迴調用，我們只存儲當前字符串的狀態和計數器。

結論​

「產生括號組合」是一個迷人的問題，不僅測試你對遞迴和回溯法的理解，還有如何有效地修剪搜索空間。關鍵的帶走點是要始終考慮在遞迴中添加元素到你的組合或排列的條件，因為這可以顯著影響你的解決方案的效率。
